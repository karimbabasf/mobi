# Mobi v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A native macOS menu-bar app that shows every AI agent running on the Mac and every payment it makes (payments mocked in v1), with a working allow/deny approval flow, in an aesthetic toolbar dropdown.

**Architecture:** Tauri 2 (Rust backend, vanilla-TypeScript + Vite frontend). The Rust side owns a single in-memory Store behind a Mutex, fed by three producers (Agent Monitor via `sysinfo`, Mock Payment Sensor, Gate) and read by the frontend over Tauri commands + a `roster-updated` event. The frontend is a hand-written `render(roster)` over ported mockup CSS, shown in a borderless vibrancy panel anchored under a tray icon.

**Tech Stack:** Tauri 2, Rust (serde, sysinfo, window-vibrancy, tokio, chrono), Vite + TypeScript (no framework), `@resvg/resvg-js` for icon generation.

## Global Constraints

- Target platform: macOS only (menu-bar app, `ActivationPolicy::Accessory`, no dock icon).
- No custody: Mobi never holds or generates a private key. Wallet registry stores addresses only. (No wallet code lands in v1 at all.)
- Payments are mocked in v1: the `PaymentSensor` trait is real, only `MockSensor` is implemented. Real shim + Base watcher are v2 and MUST NOT be built here.
- No policy engine: approvals are manual per-payment. No caps, allowlists, or auto-approve.
- No AI attribution in any commit, PR, README, or comment. No `Co-Authored-By`, no "Generated with".
- No em or en dashes in any artifact (code, README, commits, comments, UI copy). Use commas, colons, parentheses.
- USDC is rendered in the UI with the glyph `◈` (matching the mockups). Agent brand dots: Claude `#D97757`, Codex/OpenAI `#e6e6e6`, Gemini `#4285F4`.
- Rust and TS models must stay field-compatible: Rust structs use `#[serde(rename_all = "camelCase")]`, TS types mirror the camelCase shape exactly.
- Public GitHub repo `karimbabasf/mobi`. Work on feature branches, merge to `main` via PR. Never expose secrets or private-project references (JB Hunting, Roman Storm, Meridian, rtk) anywhere in the repo.

```rust
use std::collections::HashMap;
use crate::model::*;

#[derive(Default)]
pub struct Store {
    agents: HashMap<String, Agent>,
    payments: Vec<PaymentEvent>,   // insertion order; dedup by id on upsert
}

impl Store {
    pub fn new() -> Self { Self::default() }

    pub fn upsert_agent(&mut self, a: Agent) { self.agents.insert(a.id.clone(), a); }
    pub fn remove_agent(&mut self, id: &str) { self.agents.remove(id); }
    pub fn agent_ids(&self) -> Vec<String> { self.agents.keys().cloned().collect() }

    pub fn upsert_payment(&mut self, p: PaymentEvent) {
        if let Some(slot) = self.payments.iter_mut().find(|x| x.id == p.id) { *slot = p; }
        else { self.payments.push(p); }
    }

    pub fn set_payment_status(&mut self, id: &str, status: PaymentStatus,
                              settled_at: Option<String>, tx_hash: Option<String>) -> bool {
        if let Some(p) = self.payments.iter_mut().find(|x| x.id == id) {
            p.status = status;
            if settled_at.is_some() { p.settled_at = settled_at; }
            if tx_hash.is_some() { p.tx_hash = tx_hash; }
            true
        } else { false }
    }

    pub fn roster(&self) -> Roster {
        let mut entries: Vec<RosterEntry> = self.agents.values().map(|a| {
            let mut agent = a.clone();
            let payments: Vec<PaymentEvent> = self.payments.iter()
                .filter(|p| p.agent_id.as_deref() == Some(a.id.as_str())).cloned().collect();
            let pending = payments.iter().find(|p| p.status == PaymentStatus::Pending).cloned();
            if pending.is_some() { agent.status = AgentStatus::WaitingApproval; }
            let spent = |ps: &[PaymentEvent]| ps.iter()
                .filter(|p| matches!(p.status, PaymentStatus::Settled | PaymentStatus::Allowed))
                .map(|p| p.amount).sum::<f64>();
            let spend_session = spent(&payments);
            RosterEntry { agent, spend_today: spend_session, spend_session, pending, payments }
        }).collect();
        entries.sort_by(|a, b| a.agent.name.cmp(&b.agent.name));
        let live_count = entries.iter().filter(|e| e.agent.status != AgentStatus::Idle).count();
        let pending_count = entries.iter().filter(|e| e.pending.is_some()).count();
        let spend_today = entries.iter().map(|e| e.spend_today).sum();
        Roster { entries, live_count, spend_today, pending_count }
    }
}
```

(v1 has no persistence, so `spend_today == spend_session`. The two fields exist so the UI copy and the v2 daily rollover need no schema change.)

```rust
use std::collections::HashMap;
use std::sync::Mutex;
use std::time::{Duration, Instant};
use tokio::sync::oneshot;

#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub enum Decision { Allow, Deny }

#[derive(Default)]
pub struct Gate {
    pending: Mutex<HashMap<String, (Instant, oneshot::Sender<Decision>)>>,
}

impl Gate {
    pub fn new() -> Self { Self::default() }

    pub fn register(&self, payment_id: String, tx: oneshot::Sender<Decision>, requested_at: Instant) {
        self.pending.lock().unwrap().insert(payment_id, (requested_at, tx));
    }

    pub fn resolve(&self, payment_id: &str, d: Decision) -> bool {
        if let Some((_, tx)) = self.pending.lock().unwrap().remove(payment_id) {
            let _ = tx.send(d);   // receiver may have dropped; ignore
            true
        } else { false }
    }

    pub fn pending_ids(&self) -> Vec<String> {
        self.pending.lock().unwrap().keys().cloned().collect()
    }

    pub fn timeout_sweep(&self, now: Instant, ttl: Duration) -> Vec<String> {
        self.pending.lock().unwrap().iter()
            .filter(|(_, (at, _))| now.duration_since(*at) >= ttl)
            .map(|(id, _)| id.clone()).collect()
    }
}
```

(v1 timeout action is "hold": `timeout_sweep` reports overdue ids for a UI badge but does not auto-deny. The default per spec is hold, configurable later.)

```rust
/// Emits a synthetic pending payment for a live agent every few seconds,
/// routes it through the Gate (the same path the real shim will use),
/// then settles or drops it based on the user's decision.
pub async fn run_mock_sensor(store: Arc<Mutex<Store>>, gate: Arc<Gate>, app: AppHandle) {
    let mut n = 0u64;
    loop {
        tokio::time::sleep(Duration::from_secs(6)).await;
        let agent_id = { store.lock().unwrap().agent_ids().into_iter().next() };
        let Some(agent_id) = agent_id else { continue };
        n += 1;
        let id = format!("mock-{n}");
        let service = mock_service_pool()[(n as usize) % mock_service_pool().len()];
        let amount = [0.02, 0.05, 0.12, 0.20][(n as usize) % 4];
        let pay = synth_payment(&id, Some(agent_id.clone()), service, amount);
        store.lock().unwrap().upsert_payment(pay);
        let _ = app.emit("roster-updated", ());
        let (tx, rx) = tokio::sync::oneshot::channel();
        gate.register(id.clone(), tx, Instant::now());
        match rx.await {
            Ok(Decision::Allow) => {
                store.lock().unwrap().set_payment_status(&id, PaymentStatus::Settled,
                    Some(now_rfc3339()), Some(format!("0xmock{n:064x}")));
            }
            _ => { store.lock().unwrap().set_payment_status(&id, PaymentStatus::Denied, None, None); }
        }
        let _ = app.emit("roster-updated", ());
    }
}
```

```rust
/// Scans running processes for known agent CLIs. Best-effort: name + pid + start time.
pub fn scan_agents(sys: &System) -> Vec<Agent> {
    let mut out = Vec::new();
    for (pid, proc_) in sys.processes() {
        let name = proc_.name().to_string_lossy();
        if let Some((tool, display)) = classify_tool(&name) {
            let cwd = proc_.cwd().and_then(|p| p.file_name().map(|s| s.to_string_lossy().to_string()))
                .unwrap_or_else(|| "terminal".into());
            out.push(Agent {
                id: format!("proc-{}", pid), name: display.into(), tool: tool.into(),
                terminal: cwd, status: AgentStatus::Working, current_action: "active".into(),
                started_at: now_rfc3339(), wallet_addresses: vec![],
            });
        }
    }
    out
}

/// Replaces the agent set in the store. Falls back to demo agents when nothing real runs,
/// so the UI is never empty for a first look.
pub fn refresh_agents(store: &Arc<Mutex<crate::state::Store>>, sys: &mut System) {
    sys.refresh_processes(sysinfo::ProcessesToUpdate::All, true);
    let mut agents = scan_agents(sys);
    if agents.is_empty() { agents = demo_agents(); }
    let mut s = store.lock().unwrap();
    let live: std::collections::HashSet<String> = agents.iter().map(|a| a.id.clone()).collect();
    for id in s.agent_ids() { if !live.contains(&id) { s.remove_agent(&id); } }
    for a in agents { s.upsert_agent(a); }
}
```
